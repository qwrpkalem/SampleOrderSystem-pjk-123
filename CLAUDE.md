# 프로젝트 목표

반도체 시료 관리 시스템을 만든다.

# 요구사항 문서

- 전체 개요 및 도메인 모델: `docs/PRD.md`
- 기능별 상세 요구사항: `docs/features/` (메인 메뉴, 시료 관리, 시료 주문, 주문 승인/거절, 모니터링, 생산 라인, 출고 처리, 데이터 저장/불러오기)
- 원본 과제 명세: `Semi.pdf`

# 문서 동기화 규칙

- 요구사항, 정책, 비즈니스 규칙에 변경/결정 사항이 생기면 반드시 관련 문서(`docs/PRD.md`,
  `docs/features/*.md`)를 함께 Update한다. 코드/규칙만 바꾸고 문서를 갱신하지 않은 상태로
  남기지 않는다.
- 여러 문서에 걸쳐 있는 내용(예: 생산 라인과 영속성)은 관련된 모든 문서를 함께 확인하여
  일관성을 유지한다.
