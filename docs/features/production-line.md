# 생산 라인

> 출처: `Semi.pdf` Chapter 01/02 - 시스템 개요, 생산라인

## 개요

- 생산 라인은 공장에서 시료 하나를 생산하는 설비 흐름이다.
- 하나의 생산 라인은 시료를 하나씩 생산한다.
- 주문이 들어온 시료에 대해서만 생산한다.
- 생산 라인에 대한 정보를 Display하며, 주문량에 대한 부족분을 생산하되 수율 및 오차를 고려하여
  시료를 생산한다.

## 기능 요구사항

### 생산 처리

- 실 생산량 = `ceil(부족분 / 수율)`
- 총 생산 시간 = `평균 생산시간 * 실 생산량`
- 생산 완료 시 주문 상태를 `PRODUCING` → `CONFIRMED`로 변경한다.
  (상태 흐름 상세: [order-approval.md](./order-approval.md))

### 생산 현황 표기

- 현재 생산 중인 시료에 대한 정보를 표기한다.
- 표기할 정보 수준은 자율적으로 결정한다 (예: 주문 정보, 현재까지의 생산량 등).

### 대기 주문 확인

- 생산 라인의 대기열인 생산 큐를 이용한다.
- 생산 작업을 대기하고 있는 목록을 출력한다.
- 스케줄링 전략: **FIFO**.

## 비즈니스 규칙 요약

| 규칙 | 계산식 |
| --- | --- |
| 실 생산량 | `ceil(부족분 / 수율)` |
| 총 생산 시간 | `평균 생산시간 * 실 생산량` |
| 생산 큐 스케줄링 | FIFO |

## 결정 사항

- 라인개수는 단 1개 추가나 삭제 없음.
- 생산 도중 프로그램이 종료되어도, 재시작 시점에 이미 생산 완료 예정 시각이 지났다면 그 시점에
  재고 Update가 반영되어야 한다 (프로그램이 꺼져 있던 시간도 생산 경과 시간에 포함).
  데이터 영속성 방식은 [persistence.md](./persistence.md) 참고.
- 재고 Update는 생산이 진행되는 동안 실시간으로 반영하지 않고, 생산이 완료된 시점에 한 번에
  반영한다.
- 수율은 실 생산량 계산(`ceil(부족분 / 수율)`)에만 반영되며, 생산을 시도한 물량 중 실패하여
  폐기되는 것은 없다. 즉, 생산을 시도한 수량은 모두 재고에 포함된다.
- "평균 생산시간"의 단위는 **분(minute)** 이다. 문서에 단위가 명시되어 있지 않아 구현 시
  결정했다.
- 생산 완료 여부(완료 시각이 지났는지)는 프로그램 재시작 시점뿐 아니라, **콘솔 앱이 계속
  실행 중인 동안에도 메인 메뉴로 돌아올 때마다** 검사해 반영한다. 그렇지 않으면 프로그램을
  껐다 켜야만 생산 완료가 반영되는 문제가 있어 구현 시 결정했다.
