# 주문 승인/거절

> 출처: `Semi.pdf` Chapter 02 - 주문 승인/거절

## 개요

접수된 주문(`RESERVED`) 목록을 확인하고, 특정 주문에 대하여 승인 혹은 거절을 처리한다.

## 주문 상태 흐름

| 상태 | 의미 |
| --- | --- |
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (정상 흐름 외 상태, 모니터링에서 제외) |
| PRODUCING | 주문 승인 완료 및 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료 및 출고 대기 중 |
| RELEASE | 출고 완료 |

전이 규칙:

- `RESERVED` → (승인, 재고 충분) → `CONFIRMED`
- `RESERVED` → (승인, 재고 부족) → `PRODUCING` (생산 라인에 자동 등록)
- `RESERVED` → (거절) → `REJECTED`
- `PRODUCING` → (생산 완료) → `CONFIRMED` ([production-line.md](./production-line.md) 참조)
- `CONFIRMED` → (출고 실행) → `RELEASE` ([release.md](./release.md) 참조)

## 기능 요구사항

### 접수된 주문 목록

- `RESERVED` 상태의 주문 목록을 Display.

### 주문 승인

- 접수된 특정 주문에 대해 승인한다.
- 승인 시 재고 상황에 따라 2가지 방식으로 자동 처리된다.
  - 재고가 충분한 경우 → 주문을 즉시 `CONFIRMED` 상태로 전환
  - 재고가 부족한 경우 → 생산 라인에 자동으로 등록, 주문 상태를 `PRODUCING`으로 전환

### 주문 거절

- 접수된 특정 주문에 대해 거절한다.
- 즉시 `REJECTED` 상태로 전환.

## 결정 사항

- 여유는 주문대비 재고 충분 상태이고, 부족은 주문대비 재고 수량 부족 상태이다. 고갈은 수량이 0인 상태이다. (1개라도 많으면 여유, 1개라도 부족하면 부족이다.)
- 재고는 **출고(RELEASE) 시점에만** 감소한다. 승인(`CONFIRMED`/`PRODUCING` 전환) 시점에는
  재고를 전혀 변경하지 않는다. 실제로 창고에서 시료가 빠져나가는 시점은 출고 시점이므로, 그
  전까지는 재고 수량이 그대로 유지되어야 한다는 요구사항을 반영했다.
- 재고를 차감하지 않는 대신, 같은 시료에 대해 먼저 승인된 주문들이 재고를 **선점**하고 있다는
  개념으로 신규 주문의 충분/부족 여부를 판정한다.
  - 선점 수량(committed) = 같은 시료에 대해 이미 `PRODUCING` 또는 `CONFIRMED` 상태인
    다른 주문들의 주문 수량 합.
  - 가용 재고(available) = `max(0, 현재 재고 - 선점 수량)`. 선점 수량이 현재 재고보다 많아도
    가용 재고가 음수가 되지는 않는다 (그 초과분은 이미 앞선 주문의 생산으로 채워질 몫이며,
    이번 주문이 추가로 책임질 부족분이 아니다).
  - 가용 재고 >= 이번 주문 수량 → `CONFIRMED`로 전환 (재고 변경 없음).
  - 가용 재고 < 이번 주문 수량 → 부족분 = 이번 주문 수량 - 가용 재고, 그만큼 생산 라인에
    등록하고 `PRODUCING`으로 전환 (재고 변경 없음).
  - 예시: 재고 50개에 주문 A(100개) → 주문 B(50개) 순서로 들어와 순서대로 승인하는 경우
    - A 승인: 선점 0, 가용 50, 부족분 50(=100-50) → 재고 50개는 그대로 두고 부족분 50개를
      생산한다.
    - B 승인: 선점 100(=A의 주문 수량), 가용 `max(0, 50-100) = 0`, 부족분 50(=50-0) → B도
      자신의 주문 수량 50개 전부를 추가로 생산해야 한다 (재고 50개는 이미 A가 선점했기 때문).
  - 생산 완료 후 재고 반영은 [production-line.md](./production-line.md), 출고 시 재고 차감은
    [release.md](./release.md) 참조.
