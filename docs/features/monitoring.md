# 모니터링

> 출처: `Semi.pdf` Chapter 02 - 모니터링

## 개요

주문에 따른 시료 재고에 대한 정보를 모니터링하는 기능. 담당자가 현재 시스템의 상태를 한눈에
파악할 수 있도록 구성한다.

## 기능 요구사항

### 주문량 확인

- 현재 상태별(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE`) 목록을 확인한다.
- `REJECTED`는 유효한 주문이 아니므로 집계에서 제외한다.

### 재고량 확인

- 각 시료별 현재 재고 수량을 확인한다.
- 주문 대비 재고 수량에 따라 상태를 표기한다.
  - 여유: 주문 대비 재고 충분 상태
  - 부족: 주문 대비 재고 수량 부족 상태
  - 고갈: 수량이 0인 상태

## 결정 사항

- 여유는 주문대비 재고 충분 상태이고, 부족은 주문대비 재고 수량 부족 상태이다. 고갈은 수량이 0인 상태이다. (1개라도 많으면 여유, 1개라도 부족하면 부족이다.)
- 재고는 **출고(RELEASE) 시점에만** 감소한다 (참고: [order-approval.md](./order-approval.md)).
  따라서 "주문 대비"의 기준이 되는 주문 수량은 해당 시료의 **`RESERVED` + `PRODUCING` +
  `CONFIRMED`** 주문 수량의 합으로 정의한다. 이 세 상태는 모두 아직 출고되지 않아 현재 재고에
  대한 청구권을 갖고 있는 상태이기 때문이다. `RELEASE`는 이미 재고가 차감 반영된 상태이므로
  제외하고, `REJECTED`는 유효한 주문이 아니므로 제외한다.
