# 모니터링

> 출처: `Semi.pdf` Chapter 02 - 모니터링

## 개요

주문에 따른 시료 재고에 대한 정보를 모니터링하는 기능. 담당자가 현재 시스템의 상태를 한눈에
파악할 수 있도록 구성한다.

## 기능 요구사항

### 주문량 확인

- 현재 상태별(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE`) 목록을 확인한다.
- `REJECTED`는 유효한 주문이 아니므로 집계에서 제외한다.

### 재고량 확인

- 각 시료별 현재 재고 수량을 확인한다.
- 주문 대비 재고 수량에 따라 상태를 표기한다.
  - 여유: 주문 대비 재고 충분 상태
  - 부족: 주문 대비 재고 수량 부족 상태
  - 고갈: 수량이 0인 상태

## 결정 사항

- 여유는 주문대비 재고 충분 상태이고, 부족은 주문대비 재고 수량 부족 상태이다. 고갈을 수량이 0인 상태이다. (1개라도 많으면 여유, 1개라도 부족하면 부족이다.)
- "주문 대비"의 기준이 되는 주문 수량은 해당 시료의 **`RESERVED`(접수, 아직 승인되지 않음)**
  주문 수량의 합으로 정의한다. `CONFIRMED`/`RELEASE`는 이미 재고가 차감되어 반영된 상태이고,
  `PRODUCING`은 이미 생산으로 부족분이 해결 중이므로, 현재 재고의 충분/부족 판정에는
  아직 처리되지 않은 `RESERVED` 수요만 관련이 있다.
