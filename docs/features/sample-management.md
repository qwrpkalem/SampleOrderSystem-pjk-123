# 시료 관리

> 출처: `Semi.pdf` Chapter 02 - 시료 관리

## 개요

시료(Sample)는 이 시스템의 가장 기본이 되는 단위. 각 시료는 고유한 이름과 속성을 가지며,
시스템에 등록된 시료만 주문 가능하다.

## 도메인 모델 - 시료 (Sample)

| 속성 | 설명 |
| --- | --- |
| 시료 ID | 고유 식별자 |
| 이름 | 시료 이름 |
| 평균 생산시간 | 1개 생산에 소요되는 평균 시간 |
| 수율 | 정상적인 시료 / 총 생산 시료 (예: 100개 중 정상 90개 → 0.9) |
| 재고 수량 | 현재 보유 중인 정상 시료 수량 |

## 기능 요구사항

### 시료 등록

- 새로운 시료를 시스템에 추가한다.
- 입력 속성: 시료 ID, 이름, 평균 생산시간, 수율, 재고 수량.
- 이미 등록된 시료 ID로 다시 등록을 시도하면 거부한다.

### 시료 조회

- 등록된 모든 시료 목록을 확인한다.
- 현재 재고 수량도 함께 표시한다.

### 시료 검색

- 이름 등 속성으로 특정 시료를 검색한다.

## 비즈니스 규칙

- 수율 = 정상 시료 수 / 총 생산 시료 수

## 결정 사항

- 시료 등록 시 초기 재고 수량은 강제로 0에서 시작하지 않고, 등록 시 입력받은 값을 그대로
  사용한다.
- 시료 ID는 유일해야 하며, 중복 등록은 허용하지 않는다.
