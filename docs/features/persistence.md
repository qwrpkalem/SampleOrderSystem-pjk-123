# 데이터 저장/불러오기 (영속성)

## 개요

콘솔 프로그램은 종료 후에도 시료, 주문, 생산 라인 상태를 유지해야 한다. 프로그램 시작 시
저장된 데이터를 불러오고, 상태 변경 시 파일에 저장한다.

## 결정 사항

- 데이터는 **JSON** 형태의 파일로 저장하고, 프로그램 시작 시 해당 파일을 불러온다.
- 저장 대상에는 시료 목록/재고, 주문 목록과 상태, 생산 큐/진행 중인 생산 정보(생산 완료
  예정 시각 포함)가 포함되어야 한다.
  - 생산 완료 예정 시각을 저장해두어야, 프로그램이 꺼져 있던 동안에도 생산이 완료되었는지
    재시작 시점에 판단할 수 있다 (참고: [production-line.md](./production-line.md)의
    "생산 도중 프로그램 종료" 결정 사항).

## 구현 순서

- Phase 1(기반 구조)에서는 시료 목록과 주문 목록(상태 포함)만 저장/불러오기를 구현한다.
  생산 큐/진행 중인 생산 정보(생산 완료 예정 시각 포함) 저장은 해당 도메인 모델이 정의되는
  Phase 5(생산 라인)에서 함께 구현한다 (참고: `docs_temp/phase5-production-line.md`).
