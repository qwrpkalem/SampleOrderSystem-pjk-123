# Phase 8: 메인 메뉴 (콘솔 UI 통합)

## 목표

Phase 2~7에서 구현한 기능들을 콘솔 기반 메뉴로 통합해 담당자가 명령을 입력해 조작할 수 있게 한다.

## 참고 문서

- [mainmenu.md](../docs/features/mainmenu.md)

## 구현 범위

1. 기능별 선택 화면(시료 관리 / 주문 / 모니터링 / 출고 처리 / 생산 라인) 표시 및 라우팅
2. 전체 시료 요약 정보 표시 (등록 시료 종류, 총 재고, 전체 주문건수, 생산라인 상태 -
   [mainmenu.md](../docs/features/mainmenu.md) 결정 사항 참고)
3. 프로그램 시작 시 저장된 데이터 로드, 종료 시 저장 (Phase 1 영속성 계층 연동) — `AppContext`로
   완료 (참고: 아래 진행 상황)

## 진행 상황 (완료)

- `AppContext`: 시작 시 `Repository::load()` → 각 서비스 복원 → 재시작 복구까지 자동 수행,
  `save()`로 종료 시 저장
- `Summary`(`buildSummary`): 요약 정보 4개 항목(등록 시료 종류/총 재고/전체 주문건수/생산라인
  상태=대기 작업 수) 계산 및 `ConsoleUI` 메인 메뉴 화면에 출력
- `ConsoleUI`: 메인 메뉴 + 5개 하위 메뉴(시료 관리/주문/모니터링/출고 처리/생산 라인) 콘솔
  입출력 루프. `SampleOrderSystem/main.cpp`에서 `AppContext`와 연결해 실제 실행 파일로
  전체 시나리오 및 재시작 시 데이터 복원까지 수동 검증 완료
- 한글 문자열 리터럴로 인한 codepage 949 컴파일 오류를 해결하기 위해 3개 프로젝트에 `/utf-8`
  컴파일러 옵션 추가

## 완료 기준 (체크리스트)

- [x] 각 메뉴 진입/뒤로 가기 등 콘솔 흐름 테스트 작성 및 통과 (해피 패스 + 검색/거절/재고
  부족 승인 분기)
- [x] 요약 정보 표시 항목이 결정 사항과 일치하는지 테스트 작성 및 통과
- [x] 프로그램 시작/종료 시 데이터 로드/저장이 정상 동작하는지 통합 테스트 작성 및 통과
  (`AppContext`)
- [x] 전체 시나리오(등록 → 주문 → 승인 → 출고 → 모니터링 → 생산 라인) End-to-End 테스트 작성
  및 통과, 실제 실행 파일로도 수동 검증
