# Phase 3: 시료 주문 접수

## 목표

고객의 요청을 받아 `RESERVED` 상태의 주문을 생성한다.

## 참고 문서

- [order.md](../docs/features/order.md)

## 구현 범위

1. **시료 예약**: 시료 ID, 고객명, 주문 수량을 입력받아 주문 생성
2. 생성된 주문의 초기 상태는 `RESERVED`
3. 등록되지 않은 시료 ID로 주문 시도 시 거부 (결정 완료, 참고: `order.md` 결정 사항)

## 고려 사항

- `OrderBook`은 `SampleCatalog`를 참조로 받아 존재 여부만 확인한다. `Repository`와의 연동
  (생성된 주문의 영속화)은 `SampleCatalog`와 마찬가지로 아직 별도 컴포넌트로 연결되지 않았으며,
  메인 메뉴 통합(Phase 8) 또는 별도 연동 컴포넌트에서 처리한다.

## 완료 기준 (체크리스트)

- [x] 정상적인 예약 생성 시 상태가 `RESERVED`인지 테스트 작성 및 통과
- [x] 존재하지 않는 시료 ID로 주문 시도 시 동작에 대한 테스트 작성 및 통과
- [ ] 생성된 주문이 저장소에 영속화되는지 검증 (연동 컴포넌트 또는 Phase 8에서 진행)
