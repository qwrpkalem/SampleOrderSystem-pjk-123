# Phase 3: 시료 주문 접수

## 목표

고객의 요청을 받아 `RESERVED` 상태의 주문을 생성한다.

## 참고 문서

- [order.md](../docs/features/order.md)

## 구현 범위

1. **시료 예약**: 시료 ID, 고객명, 주문 수량을 입력받아 주문 생성
2. 생성된 주문의 초기 상태는 `RESERVED`
3. 등록되지 않은 시료 ID로 주문을 시도하는 경우에 대한 처리 (문서에 명시 없음 → 구현 시 정책 결정
   필요, 결정 시 `order.md`에 반영)

## 완료 기준 (체크리스트)

- [ ] 정상적인 예약 생성 시 상태가 `RESERVED`인지 테스트 작성 및 통과
- [ ] 존재하지 않는 시료 ID로 주문 시도 시 동작에 대한 테스트 작성 및 통과 (정책 결정 후)
- [ ] 생성된 주문이 저장소에 영속화되는지 테스트 작성 및 통과
