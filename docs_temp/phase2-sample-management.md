# Phase 2: 시료 관리 (등록 / 조회 / 검색)

## 목표

시료를 시스템에 등록하고, 목록을 조회하거나 이름으로 검색할 수 있게 한다.

## 참고 문서

- [sample-management.md](../docs/features/sample-management.md)

## 구현 범위

1. **시료 등록**: 시료 ID, 이름, 평균 생산시간, 수율, 재고 수량을 입력받아 새로운 시료를 추가
2. **시료 조회**: 등록된 모든 시료 목록과 현재 재고 수량을 함께 표시
3. **시료 검색**: 이름 등 속성으로 특정 시료를 검색

## 고려 사항

- 시료 등록 시 초기 재고 수량 처리 방식 결정 완료: 등록 시 입력받은 값을 그대로 사용한다
  (참고: [sample-management.md](../docs/features/sample-management.md#결정-사항)).
- `SampleCatalog`는 현재 메모리 상의 목록만 다루며, `Repository`와의 연동(등록한 시료의
  영속화)은 아직 별도 컴포넌트로 연결되지 않았다. 메인 메뉴 통합(Phase 8) 또는 별도
  연동 컴포넌트에서 처리한다.

## 완료 기준 (체크리스트)

- [x] 시료 등록 성공/중복 ID 등록 시도에 대한 테스트 작성 및 통과
- [x] 시료 조회(전체 목록) 테스트 작성 및 통과
- [x] 이름 기반 검색(부분 일치) 테스트 작성 및 통과
- [x] 초기 재고 수량 정책 결정 및 관련 문서 Update
- [ ] `Repository`와 연동해 등록한 시료가 실제로 영속화되는지 검증 (연동 컴포넌트 또는
  Phase 8에서 진행)
